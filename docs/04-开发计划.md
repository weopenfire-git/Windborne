## 3. P0 验收清单

### 3.1 功能验收

- [ ] 用户可注册、登录、登出
- [ ] 用户可修改头像、昵称、简介
- [ ] 用户可新增、编辑、删除飞行日志
- [ ] 飞行日志包含纸质版所有字段
- [ ] 用户可上传多张机票图片
- [ ] 用户可将单条日志设为公开/私有
- [ ] 统计仪表盘数据正确（次数/时长/距离/机场数）
- [ ] 飞行地图正确显示航线
- [ ] 广场展示所有公开日志
- [ ] 未登录用户可浏览广场
- [ ] 数据迁移工具可用（localStorage → 云端）
- [ ] 迁移后图片正确显示

### 3.2 非功能验收

- [ ] 首屏加载 ≤ 2 秒
- [ ] API 响应 ≤ 500ms
- [ ] 移动端 375px 宽度可用
- [ ] 无控制台错误
- [ ] Lighthouse Performance ≥ 80
- [ ] Lighthouse Accessibility ≥ 90
- [ ] HTTPS 强制
- [ ] RLS 策略无越权漏洞

### 3.3 安全验收

- [ ] 未登录用户无法访问 /flights
- [ ] 用户 A 无法修改用户 B 的数据
- [ ] 用户 A 无法查看用户 B 的私有日志
- [ ] SQL 注入测试通过（参数化查询）
- [ ] XSS 测试通过（输入转义）
- [ ] 图片上传类型/大小限制生效

## 5. 风险与应对

### 5.1 技术风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Supabase 免费档资源不足 | 中 | 中 | 监控使用量，接近上限时迁 MemFire Cloud |
| Cloudflare Pages 构建失败 | 低 | 中 | 本地构建验证 + Preview 环境测试 |
| RLS 策略配置错误 | 中 | 高 | 上线前用不同账号测试越权 |
| 图片上传超限 | 低 | 低 | 客户端压缩 + 服务端二次校验 |
| 数据迁移丢数据 | 中 | 极高 | 迁移前后对比记录数 + 图片数 |

### 5.2 产品风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 儿子和朋友不爱用 | 中 | 高 | P0 聚焦核心需求，快速验证 |
| 社区冷场 | 高 | 高 | 儿子做种子用户，定期发内容 |
| 国内访问慢 | 中 | 中 | Cloudflare CDN + 监控 |
| 内容质量低 | 低 | 中 | 飞友圈垂直，初期质量可控 |

### 5.3 运营风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 备案延迟 | 中 | 中 | 早期不备案，正式运营再走 |
| 数据丢失 | 低 | 极高 | 每日自动备份 + 每周手动备份 |
| 费用超支 | 低 | 低 | 全部用免费档，超出才考虑付费 |
