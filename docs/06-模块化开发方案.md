# Windborne · 模块化开发方案

> 版本：v1.0  
> 日期：2026-07-19  
> 目的：定义模块边界、接口契约、交付物规范，支持独立测试

---

## 1. 模块化原则

### 1.1 设计原则

- **单一职责**：每个模块只做一件事
- **接口契约**：模块间通过明确接口通信，不共享内部状态
- **独立可测**：每个模块可独立部署、独立测试
- **向后兼容**：接口一旦定义，新增字段不破坏旧版本
- **文档先行**：接口文档先于代码，代码实现文档

### 1.2 交付物

每个模块交付 3 样东西：

```
模块 N 交付包
├── 代码          # 实现代码 + 迁移 SQL
├── 接口文档      # tests/0N-模块名.md
└── 测试用例      # 接口文档内含测试用例
```

## 3. 各模块详细定义

### 模块 01 · 数据库

| 项 | 说明 |
|----|------|
| **边界** | 只负责数据存储 schema，不含业务逻辑 |
| **输入** | SQL 迁移文件 |
| **输出** | 可用的数据库表 + 索引 + RLS + 触发器 + 视图 + 种子数据 |
| **接口** | SQL 层面，无 HTTP 接口 |
| **交付物** | `migrations/001-007.sql` + `tests/01-database.md` |
| **验收** | 所有表可 CRUD，RLS 策略生效，种子数据正确 |

### 模块 02 · 认证

| 项 | 说明 |
|----|------|
| **边界** | 注册、登录、登出、会话管理、路由保护 |
| **输入** | 邮箱 + 密码 |
| **输出** | JWT token + 用户 session |
| **接口** | `/api/auth/register`, `/api/auth/login`, `/api/auth/logout`, `/api/auth/me` |
| **交付物** | 认证代码 + Middleware + `tests/02-auth.md` |
| **验收** | 注册创建 users 记录，登录返回 token，路由保护生效 |

### 模块 03 · 用户资料

| 项 | 说明 |
|----|------|
| **边界** | 用户资料的查看和修改，头像上传 |
| **输入** | 昵称、简介、头像文件 |
| **输出** | 更新后的用户资料 |
| **接口** | `GET /api/user/profile`, `PUT /api/user/profile`, `POST /api/user/avatar` |
| **交付物** | 用户 API + 头像上传 + `tests/03-user-profile.md` |
| **验收** | 用户可修改资料，头像上传到 Storage，username 唯一 |

### 模块 04 · 飞行日志

| 项 | 说明 |
|----|------|
| **边界** | 飞行日志的增删改查，机票图片上传 |
| **输入** | 完整的 flight 字段 + 机票图片 |
| **输出** | 持久化的 flight 记录 |
| **接口** | `GET/POST/PUT/DELETE /api/flights`, `POST /api/flights/[id]/tickets` |
| **交付物** | Flights API + 机票上传 + `tests/04-flights.md` |
| **验收** | CRUD 完整，RLS 隔离生效，图片上传到 Storage |

### 模块 05 · 统计

| 项 | 说明 |
|----|------|
| **边界** | 用户飞行数据的聚合统计 |
| **输入** | 用户 ID |
| **输出** | 次数、时长、距离、机场数、机型数等 |
| **接口** | `GET /api/stats` |
| **交付物** | 统计 API + 视图 + `tests/05-stats.md` |
| **验收** | 数据正确，空用户返回 0 |

### 模块 06 · 公开广场

| 项 | 说明 |
|----|------|
| **边界** | 展示所有公开的飞行日志，只读 |
| **输入** | 分页参数 |
| **输出** | 公开飞行日志列表 |
| **接口** | `GET /api/feed` |
| **交付物** | Feed API + `tests/06-feed.md` |
| **验收** | 未登录可访问，只返回 is_public=true 的记录 |

### 模块 07 · 数据迁移

| 项 | 说明 |
|----|------|
| **边界** | 从旧 flight-log 的 localStorage JSON 导入到云端 |
| **输入** | JSON 文件（旧数据） |
| **输出** | 写入的 flights + tickets 数量 |
| **接口** | `POST /api/import` |
| **交付物** | 导入 API + 迁移脚本 + `tests/07-import.md` |
| **验收** | JSON 解析正确，图片 base64 转存 Storage，重复数据跳过 |

### 模块 08 · 前端页面

| 项 | 说明 |
|----|------|
| **边界** | 所有用户可见的页面 |
| **输入** | 用户操作 |
| **输出** | 渲染的 HTML |
| **接口** | 页面路由（非 API） |
| **交付物** | Next.js 页面 + 组件 + `tests/08-frontend.md` |
| **验收** | 所有页面可访问，响应式，HUD 风格统一 |

## 5. 开发节奏

### 5.1 单模块开发流程

```
1. 写接口文档（tests/0N-xxx.md）
   ↓
2. 实现代码
   ↓
3. 自测（按测试用例跑一遍）
   ↓
4. 交付：代码 + 接口文档
   ↓
5. 英明转发给测试 AI
   ↓
6. 测试 AI 执行测试，反馈结果
   ↓
7. 修复（如有 FAIL）
   ↓
8. 回归测试
   ↓
9. 模块完成，进入下一个
```

### 5.2 并行策略

- **可并行**：模块 03（用户）和 04（飞行日志）在 02 完成后可并行
- **不可并行**：01 → 02 → 03/04 是严格依赖链
- **前端**：模块 08 在后端模块（01-07）完成后启动，但可提前做静态页面

### 5.3 交付频率

- 每完成一个模块立即交付（不等其他模块）
- 交付物推送到 GitHub，英明拉取后转发测试 AI
- 测试 AI 反馈后，修复优先级高于新模块开发

## 7. 测试 AI 使用说明

测试 AI 拿到一个模块后，按以下步骤工作：

1. **阅读背景**：读 `tests/README.md` 了解项目全局
2. **读接口文档**：读 `tests/0N-xxx.md` 了解本模块接口和测试用例
3. **读设计文档**（可选）：读 `docs/` 下相关文档深入了解
4. **准备环境**：确认环境变量、测试账号可用
5. **执行测试**：按测试用例逐项执行，记录结果
6. **反馈**：输出测试报告，格式：

```
## 测试报告 · 模块 0N

### 测试环境
- 时间: xxx
- 测试人: xxx
- 环境: local / staging

### 测试结果
| 用例 | 状态 | 实际结果 |
|------|------|----------|
| TC-001 | PASS | - |
| TC-002 | FAIL | 实际返回 500，预期 200 |

### 问题汇总
1. TC-002: 请求 /api/xxx 返回 500，错误信息: xxx
   复现步骤: xxx

### 结论
- 通过率: 15/16 (94%)
- 阻塞问题: 1 个
- 建议: 修复 TC-002 后重新测试
```
