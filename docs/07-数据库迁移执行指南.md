# Windborne 数据库迁移执行指南

## 概述

本文档说明如何在 Supabase 上执行数据库迁移，创建所有表、索引、触发器、视图和种子数据。

## 方式一：自动脚本（推荐，需数据库密码）

### 1. 获取数据库密码

1. 打开 [Supabase Dashboard](https://supabase.com)
2. 进入你的 `windborne` 项目
3. 左侧边栏 → **Settings** → **Database**
4. 找到 **Connection string**，格式类似：
   ```
   postgresql://postgres:<PASSWORD>@db.cjdjtapzkjambtawbfiz.supabase.co:5432/postgres
   ```
5. 复制 `<PASSWORD>` 部分（不含 `<>`）

### 2. 设置环境变量

```bash
export DATABASE_URL="postgresql://postgres:<你的密码>@db.cjdjtapzkjambtawbfiz.supabase.co:5432/postgres"
```

### 3. 执行迁移

```bash
cd /path/to/flight-log/web
npx ts-node scripts/migrate.ts
```

或者直接用 `psql`（如果已安装）：

```bash
PGPASSWORD=<你的密码> psql \
  -h db.cjdjtapzkjambtawbfiz.supabase.co \
  -p 5432 \
  -U postgres \
  -d postgres \
  -f ../migrations/full_migration.sql
```

## 方式二：Supabase SQL Editor（手动，无需密码）

### 1. 打开 SQL Editor

1. 进入 Supabase Dashboard → 左侧边栏 → **SQL Editor**
2. 点击 **New query** 或 **+**

### 2. 粘贴 SQL

1. 打开本地文件 `migrations/full_migration.sql`
2. 全选复制（Cmd+A → Cmd+C）
3. 粘贴到 SQL Editor
4. 点击 **Run**（右上角绿色按钮）

### 3. 验证结果

执行完成后，左侧 **Tables** 面板应该显示：
- `users`
- `flights`
- `flight_tickets`
- `posts`
- `comments`
- `likes`
- `follows`
- `airports`
- `aircraft`

以及视图：
- `user_stats`
- `public_feed`
- `airport_stats`
- `aircraft_stats`

---

## 方式三：分步执行（如果完整 SQL 太大）

如果 SQL Editor 执行完整文件超时，可以分步执行：

1. 执行 `migrations/001_init_tables.sql` → 创建表
2. 执行 `migrations/002_init_indexes.sql` → 创建索引
3. 执行 `migrations/003_init_rls.sql` → 启用 RLS 和策略
4. 执行 `migrations/004_init_triggers.sql` → 创建触发器
5. 执行 `migrations/005_init_views.sql` → 创建视图
6. 执行 `migrations/006_seed_airports.sql` → 导入机场数据
7. 执行 `migrations/007_seed_aircraft.sql` → 导入机型数据

---

## 验证迁移

执行完成后，运行以下 SQL 验证：

```sql
-- 检查表数量
SELECT tablename FROM pg_tables WHERE schemaname = 'public';

-- 检查视图数量
SELECT viewname FROM pg_views WHERE schemaname = 'public';

-- 检查机场数据
SELECT COUNT(*) FROM airports;

-- 检查机型数据
SELECT COUNT(*) FROM aircraft;

-- 检查 RLS 策略
SELECT tablename, policyname FROM pg_policies WHERE schemaname = 'public';
```

---

## 安全提示

- **数据库密码** 拥有数据库全部权限，执行后建议立即在 Supabase 中重置
- **service_role key** 不要泄露给任何人，只用于后端管理
- **publishable key** 可以公开，前端使用
- 执行迁移后，建议删除或重置数据库密码
