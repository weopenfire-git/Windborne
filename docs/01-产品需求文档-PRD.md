## 1. 项目背景

### 1.1 起源

英明儿子是资深飞友（aviation enthusiast），长期用纸质飞行日志记录每次行程，包括航班信息、V1/Vr/V2 速度、离场到达程序、乘务组留言、票据粘贴等字段。已有一份驾驶舱 HUD 风格的飞行日志网站（flight-log），数据存在浏览器 localStorage。

### 1.2 升级动机

- **数据安全**：localStorage 数据随设备丢失，需要云端持久化
- **社交需求**：儿子希望和飞友朋友交流飞行体验、分享拍机照片
- **内容沉淀**：飞友圈缺乏一个集「个人记录 + 社区交流 + 知识沉淀」于一体的轻量平台

### 1.3 项目定位

**Windborne 不是传统版块式论坛，而是「个人飞行日志 + 动态广场 + 知识库」三层结构的飞友社区。** 工具属性拉人进来，社区属性留住人，知识库形成长期资产。

## 2. 目标用户

### 2.1 核心用户

| 用户画像 | 占比 | 特征 |
|----------|------|------|
| 飞友（拍机爱好者） | 60% | 关注机型、注册号、航司，常去机场拍机 |
| 频繁出差旅客 | 25% | 关心舱位、里程、积分、机场体验 |
| 航空从业者 | 10% | 飞行员、乘务、空管，专业内容贡献者 |
| 航空知识爱好者 | 5% | 学习型用户，提问为主 |

### 2.2 用户场景

- **场景 A**：儿子飞完一趟，想记录航程并分享给飞友朋友看
- **场景 B**：飞友在机场拍到稀有机型，想发照片和位置让圈内人看到
- **场景 C**：新人想查某机型的座椅布局、某机场的贵宾室体验
- **场景 D**：用户想回顾自己一年的飞行轨迹和里程统计

## 3. 产品形态

### 3.1 三层结构

```
┌─────────────────────────────────────────────┐
│  ③ 知识库 · 集体沉淀层                       │
│  机型百科 / 机场攻略 / 航司点评 / 航线推荐    │
│  由用户内容自动聚合 + 人工编辑维护            │
├─────────────────────────────────────────────┤
│  ② 动态广场 · 公开交流层                     │
│  飞行报告 / 拍机照片 / 体验点评 / 提问答疑    │
│  类似小红书 + 论坛的混合形态                  │
├─────────────────────────────────────────────┤
│  ① 个人飞行日志 · 私有工具层                  │
│  航程记录 / 机票 / 统计 / 地图                │
│  默认私有，可选公开分享到广场                  │
└─────────────────────────────────────────────┘
```

### 3.2 设计原则

- **工具优先**：飞行日志是核心抓手，社区是衍生价值
- **小而精**：飞友圈小，重质量不重数量
- **渐进公开**：用户自主选择公开内容，降低社交压力
- **移动友好**：飞友常在机场用手机，移动端体验优先

## 5. 非功能需求

### 5.1 性能

- 首屏加载 ≤ 2 秒（国内 Cloudflare CDN）
- API 响应 ≤ 500ms
- 图片上传 ≤ 5 秒（单张 ≤ 5MB）
- 支持 100 并发用户（MVP 阶段）

### 5.2 安全

- 所有数据通过 HTTPS 传输
- 密码 bcrypt 加密（Supabase Auth 默认）
- RLS 行级权限：用户只能修改自己的数据
- 图片上传限制类型（jpg/png/webp）和大小（≤ 5MB）
- 防垃圾：注册后 24 小时内限发 5 帖

### 5.3 可访问性

- 移动端优先响应式设计
- 支持键盘导航
- 图片 alt 文本
- 色彩对比度 ≥ 4.5:1

### 5.4 SEO

- 公开页面 SSR 渲染
- 结构化数据（Flight schema）
- sitemap.xml 自动生成
- 机型/机场聚合页可被搜索引擎收录

## 6. 用户故事

### 6.1 P0 用户故事

**US-01：新用户注册**
> 作为一名飞友，我想注册账号，这样我就能在云端保存我的飞行日志，换设备也能访问。

**US-02：导入旧数据**
> 作为现有 flight-log 用户，我想把浏览器里的飞行记录一键导入云端，这样我之前的记录不会丢失。

**US-03：记录新航程**
> 作为飞友，我想添加一条新的飞行日志，记录航班号、机型、座位、V1/Vr/V2 等专业字段，并上传机票照片。

**US-04：公开分享**
> 作为飞友，我想把某条特别精彩的飞行日志设为公开，让其他飞友能在广场看到。

**US-05：浏览广场**
> 作为访客，我想不登录就能浏览公开的飞行日志，这样我能决定是否注册。

### 6.2 P1 用户故事

**US-06：发帖交流**
> 作为飞友，我想在广场发帖分享拍机照片和体验，并@提到同好一起讨论。

**US-07：评论互动**
> 作为飞友，我想对别人的飞行日志或帖子评论，表达我的看法或提问。

## 8. 成功指标

### 8.1 P0 验收标准

- 儿子和 ≥ 3 个朋友完成注册
- 至少导入 20 条历史飞行日志
- 7 日留存率 ≥ 50%

### 8.2 P1 验收标准

- 日活用户 ≥ 10
- 日均发帖 ≥ 3 条
- 帖子互动率（评论+点赞/浏览）≥ 10%

### 8.3 长期目标（6 个月）

- 注册用户 ≥ 500
- 月活用户 ≥ 100
- 知识库沉淀 ≥ 50 个机型/机场页面

## 9. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 国内访问不稳定 | 中 | 高 | 先用 Cloudflare，严重时迁 MemFire Cloud |
| 社区冷场 | 高 | 高 | 儿子和朋友先做种子用户，定期发内容 |
| 图片存储超限 | 低 | 中 | 免费档 1GB，超限后压缩或迁 Cloudflare R2 |
| 备案延迟 | 中 | 中 | 早期不备案，正式运营时再走流程 |
| 数据丢失 | 低 | 极高 | Supabase 自动备份 + 每周手动导出 |
